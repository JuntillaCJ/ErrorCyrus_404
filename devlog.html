Entry 1 - 2025/03/31

The first high-level language that I messed around was JavaScript. When I then hopped on python, I noticed the naming conventions of variables and functions were different. In JavaScript, the convention is to use camel case for variables and functions, where the first letter of each word is capitalized except for the first word. For example, getElementById() and myVariable. In python, the convention is to use underscore to seperate words, like my_function and name_list.

Entry 2 - 2025/06/30

This particular video is an eye opener to iteration, which is a core aspect in programming. In particular, it changed how I view iterations in python vs other languages. In other languages, you typically initialize an i variable for the index. In python, you actually iterate over an iterable directly. Compare the two:


  for i in range(len(names)):
      print(names[i])
      
  for name in names:
      print(name)

The pythonic iteration is just more direct. Another case where you want a numbered list:


  for i in range(len(names)):
      print(i+1, names[i])
      
  for idx, name in enumerate(names, start=1):
      print(idx, name)

Less thinking required because you get the variables directly rather than using i to get them.

Entry 3 - 2025/07/31

Object-oriented programming is great. It effectively lets you create custom data types with their own attributes and methods. I however learned from this video that sometimes, it is completely unnecessary, and will just take up extra space. Taken from when I was making Blackjack:


  class Card:
      def __init__(self, rank, suit):
          self.rank = rank
          self.suit = suit
      
      def __str__(self):
          return f"{self.rank} of {self.suit}

The code above is perfectly fine. It defines what a card object should be, and it knows how it should display itself when you call print() on it. In this situation though, it is completely unnecessary. Notice that this class definition has two methods, one of which is init. In this case, better to just make a function out of the non-init method.


  def print_card(card):
      rank = card[0]
      suit = card[1]
      return f"{rank} of {suit}"

There are much better examples out there, especially when the non-init method is not a magic method and has an argument to pass, which can also make the usage way less elegant. So it is nice to be aware of this before I have to make code for productions.
